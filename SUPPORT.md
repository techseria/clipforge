# Getting Help

There are several ways to get help with ClipForge, depending on what you need.

Start here — most questions are answered in the docs:

- **[README.md](README.md)** — overview, quick start, features
- **[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)** — system design, data model, request flow
- **[docs/DEVELOPMENT.md](docs/DEVELOPMENT.md)** — local dev setup, debugging, testing
- **[docs/DEPLOYMENT.md](docs/DEPLOYMENT.md)** — production deploy (Docker, Kubernetes, bare metal)
- **[docs/API.md](docs/API.md)** — full REST + WebSocket reference
- **[docs/RELEASE_NOTES.md](docs/RELEASE_NOTES.md)** — what's in the current release
- **[Tasks.md](Tasks.md)** — the development task tracker (48/48 done for V1)

## 💬 Community support (free, public)

For general questions, ideas, and discussion:

- **[GitHub Discussions](https://github.com/techseria/clipforge/discussions)** — best place to ask usage questions
- **Show & tell** — share what you've built with ClipForge
- **Ideas** — propose and vote on feature ideas
- **Q&A** — get help from other users and maintainers

Maintainers check Discussions daily. Be sure to:

1. Search existing threads first
2. Provide a clear title and reproduction details
3. Include your ClipForge version (`git rev-parse HEAD` or release tag)
4. Share relevant logs and environment info

## 🐛 Bug reports

Use the [GitHub Issues](https://github.com/techseria/clipforge/issues) tracker with the **Bug Report** template. See [CONTRIBUTING.md](CONTRIBUTING.md#1-report-bugs) for what to include.
