## Becoming a maintainer

We welcome new maintainers! The path is:

1. **Be active** — submit quality PRs, review others' PRs, answer questions in Discussions, triage issues for a few months
2. **Demonstrate** — show judgment (good design decisions), reliability (follow through on commitments), and collaboration (help others)
3. **Nominated** — a current maintainer nominates you; existing maintainers vote; 2/3 majority required
4. **Onboarded** — added to the GitHub team, given merge access, paired with a senior maintainer for the first release

There is no quota or time-based promotion. We promote based on demonstrated trust.

### Maintainer responsibilities

- Triage incoming issues within 48h
- Review PRs within 3 business days
- Participate in monthly release coordination
- Uphold the [Code of Conduct](CODE_OF_CONDUCT.md)
- Be respectful, helpful, and welcoming in all community interactions

### Maintainer inactivity policy

If a maintainer has been inactive for 6+ months:

1. We'll reach out privately to ask if they want to step down or take a sabbatical
2. If no response in 2 weeks, we move them to "emeritus" status
3. They keep the @maintainer badge but lose merge access until they return
4. Returning is easy — just say "I'm back" in Discussions

## How decisions are made

- **Day-to-day** — any maintainer can merge a PR after one approval from another maintainer
- **Feature additions** — proposed in a [Discussion](https://github.com/techseria/clipforge/discussions) (Ideas category); simple majority of maintainers within 7 days
- **Breaking changes** — require all active maintainers to agree; release notes must include a migration guide
- **Security** — security team has final say; not subject to majority vote
- **Licensing** — Techseria Legal has final say

See [GOVERNANCE.md](GOVERNANCE.md) for the full governance model.
